How does it work?

The bookmarklet first downloads a reduced/compressed version of the Unihan database, which contains pinyin and frequency info for each character in the Unicode CJK Unified Ideographs. Then, using a depth-first search traversal of the DOM tree to find text nodes, the script check each character against the database. When the script finds a match, the text node is replaced by a span, with individual characters as <ruby> elements with pinyin annotations in <rt>.
